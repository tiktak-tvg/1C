# Работа с программой 1С: Бухгалтерия ред. 3.0 с применением упрощённой системы налогообложения.
#### Задача. Фирма АО «Орион» была зарегистрирована 28.09. и применяла общую систему налогообложения. 28 октября организация подала заявление о переходе на упрощенную систему налогообложения. По результатам рассмотренного заявления, организация переходит с 01.01 на упрощенную систему налогообложения. В качестве объекта налогообложения выбраны доходы, уменьшенные на величину расхода.

#### Настройк программы на учёт.
В начале ведения учета в программе пользователю необходимо, задать параметры для работы в системе, ввести справочную информацию, задать константы, ввести начальные остатки по счетам бухгалтерского учета, то есть настроить программу на учет.
#### Параметры системы.
В окне персональных настроек мы можем задать рабочую дату.

**Главное → Настройки → Персональные настройки**

Рабочая дата будет автоматически подставляться во все объекты программы. При первом запуске 1С рабочая дата считывается с компьютера. Если флажок «использовать текущую дату компьютера» сброшен, то будет учитываться дата, указанная в поле «Использовать значение». Начальные остатки вводятся на последнее число периода, предшествующего расчетного. Соответственно, устанавливаем рабочую дату 31.12.

![image](https://github.com/tvgVita69/1C/assets/98489171/ffaa699a-10bf-4c89-80d9-ca858453bc6c)

Программа будет помнить, и хранить рабочую дату, пока пользователь ее не изменит.

поступления товаров, их оплаты и реализации, что было подтверждено письмом Минфина РФ от 29 октября 2010 г. N 03-11-09/95. <br>

Флажок «списание запасов» устанавливается, только если это прописано в учётной политике организации».<br>
Если организация занимается импортом, то в разделе «Таможенные платежи» устанавливаются все флажки.<br>

В открывшемся окне выбрать двойным щелчком левой клавиши мыши группу «Товары». Табличная часть автоматически будет заполнена списком товаров. <br>

### Перечисление налогов
#### Хозяйственная операция № 29.
**15.02.** Перечислены налоги

Перечисление налогов оформляется документами «Платежное поручение» и «Списание с расчетного счета».<br>
Для быстрого формирования платежных поручений по всем налогам, подлежащим к перечислению, в программе предусмотрена обработка «Формирование платежных поручений на уплату налогов». Установим рабочую дату равной дате перечисления налогов.<br>
**Банк и касса → Банк→ Платежные поручения**

1. Установим рабочую дату, равной дате перечисления налогов.<br>
2. Нажать кнопку «Оплатить», щелкнуть по команде «Начисленные налоги и взносы». Программа автоматически определяет остатки по счетам на дату.<br>
3. В табличной части отметить строки, по которым надо сформировать платежные поручения.<br>
4. Нажать кнопку «Создать платежные документы»<br>
![image](https://github.com/tvgVita69/1C/assets/98489171/6a604f23-39bd-4595-b216-44c999b2af33)
<br>Программа сформирует платежные поручения<br>
Документ можно из окна обработки открыть двойным щелчком левой клавиши мыши, проверить, распечатать.<br>
Платежные поручения автоматически регистрируются в журнале **Банк и касса → Банк→ Платежные поручения**.<br>
Факт перечисления налогов с расчетного счета регистрируется вводом на основании созданных платежных поручений документов «Списание с расчетного счета».<br> Для быстрого формирования документов по нескольким платежным поручениям.<br>
1. Установить рабочую дату равную дате перечисления налогов.<br>
2. Открыть **Банки касса → Банк→ Банковские выписки**.<br>
3. На панели инструментов нажать кнопку «Подобрать».<br>
![image](https://github.com/tvgVita69/1C/assets/98489171/0bf0f759-fc33-45de-b625-398a9c045cff)
<br>Программа автоматически сформирует списание с расчетного счета по всем выделенным платежным документам.<br>
Сформируем отчет Оборотно-сальдовая ведомость, чтобы убедиться, что по 68 и 69 счетам отсутствует сальдо на конец периода, т.е. налоги перечислены в полном объеме.

# Работа с программой 1С: Бухгалтерия ред. 3.0 с применением упрощённой системы налогообложения.
#### Задача. Фирма АО «Орион» была зарегистрирована 28.09. и применяла общую систему налогообложения. 28 октября организация подала заявление о переходе на упрощенную систему налогообложения. По результатам рассмотренного заявления, организация переходит с 01.01 на упрощенную систему налогообложения. В качестве объекта налогообложения выбраны доходы, уменьшенные на величину расхода.

##### Настройк программы на учёт.
В начале ведения учета в программе пользователю необходимо, задать параметры для работы в системе, ввести справочную информацию, задать константы, ввести начальные остатки по счетам бухгалтерского учета, то есть настроить программу на учет.
##### Параметры системы.
В окне персональных настроек мы можем задать рабочую дату.

**Главное → Настройки → Персональные настройки**

Рабочая дата будет автоматически подставляться во все объекты программы. При первом запуске 1С рабочая дата считывается с компьютера. Если флажок «использовать текущую дату компьютера» сброшен, то будет учитываться дата, указанная в поле «Использовать значение». Начальные остатки вводятся на последнее число периода, предшествующего расчетного. Соответственно, устанавливаем рабочую дату 31.12.

![image](https://github.com/tvgVita69/1C/assets/98489171/ffaa699a-10bf-4c89-80d9-ca858453bc6c)

Программа будет помнить, и хранить рабочую дату, пока пользователь ее не изменит.

##### Сведения о собственной организации.

Проверим сведения о собственной организации вводятся в одноименном справочнике. Как заполняются сведения о собственной организации и основные справочники мы рассматривали на первом уровне.

**Главное → Настройки → Организации**

Если учет ведется «с нуля», то для ввода сведений надо нажать кнопку «Создать»

![image](https://github.com/tvgVita69/1C/assets/98489171/2d667d14-2609-40dd-9549-cf542cf522e6)

##### Настройка параметров учета.
**Администрирование → Настройки программы→ Параметры учета.**

В этом окне задаются общие параметры ведения учета для организации. Зададим параметры для фирмы «Орион». <br>Перейдем по **ссылке «Настройка плана счетов»**<br>
**Ссылка «Учет запасов»**<br>
Если включается параметр «по партиям (документам поступления)», то на счетах, учитывающих МПЗ, будет автоматически добавлен новый параметр аналитики «партии», он обязательно устанавливается, если организация на УСН с объектом налогообложения доходы минус расходы.<br>
В типовой конфигурации предусмотрено ведение складского учета только по номенклатуре. Установим ведение складского учета «по количеству и сумме».

![image](https://github.com/tvgVita69/1C/assets/98489171/fc50a0be-fd03-432f-84b7-77d21c83be69)

**Ссылка «Учет расчетов с персоналом»** дает возможность выбрать в каком разрезе будет вестись аналитический учет по 70 счету. Если, при ведении многофирменного учета», хотя бы одна организация ведет учет сводно по работникам, то включается опция «сводно по всем работникам».

![image](https://github.com/tvgVita69/1C/assets/98489171/41ce54f7-84bf-4925-a65d-3de44fca3ed0)

**Ссылка «Учет движения денежных средств»**

Установим дополнительную возможность ведения учета по статьям движения денежных средств. Обороты по статьям движения денежных средств будут использоваться при автоматическом формировании регламентированной отчетности.

![image](https://github.com/tvgVita69/1C/assets/98489171/ea11e192-a695-495c-81da-4bc8bff505c0)

**Ссылка «Учет затрат»**

![image](https://github.com/tvgVita69/1C/assets/98489171/7c16e059-edaf-40f8-a699-88d0ffd26f0c)

**По ссылке «Сроки оплаты покупателями»** установим сроки оплаты 20 дней.
Сроки оплаты покупателей, установленные в данном окне, при вводе договора с контрагентом будут предлагаться автоматически. <br>Также число дней может быть установлено в конкретном договоре с конкретным контрагентом. <br>Будет использоваться для отсчета задолженности «по сомнительным долгам». Задолженность можно отслеживать в отчетности для руководителя.

![image](https://github.com/tvgVita69/1C/assets/98489171/5e51a488-85ad-4613-b31d-3d8d0e19bc23)

**Ссылка «Настройки зарплаты»**

Так как мы будем рассматривать пример расчета зарплаты в программе 1С Бухгалтерия, то оставим включенной опцию «в этой программе». <br>Если учет ведется в другой программе, то включается опция «во внешней программе». <br>В этом случае в программе «1С Бухгалтерия» блокируется возможность введения кадровых и расчетных документов по оплате труда. Подробно настройки будут рассмотрены на теме по оплате труда.<br>
Остальные закладки оставляем без изменений.<br>
**Внимание!** При ведении «многофирменного учета» настройки распространяются на все фирмы, входящие в компанию.

#### Учетная политика.
**Главное → Настройки → Учетная политика**

Параметры определяются строго в соответствии с учетной политикой организации. С даты регистрации фирма «Орион» применяла общую систему налогообложения.

![image](https://github.com/tvgVita69/1C/assets/98489171/08b409ca-db72-4920-a26c-608a01a21020)

С 01.01. фирма «Орион» применяет упрощенную систему налогообложения. Задаем новую учетную политику.

![image](https://github.com/tvgVita69/1C/assets/98489171/d54294c9-c66e-4f9e-849d-fe68b092b4fe)

Введем учетную политику в целях налогового учета для фирмы «Орион» с упрощенной системой налогообложения на начало текущего года.<br>
**Главное → Настройки → Налоги и отчеты, ссылка «Система налогообложения».**

![image](https://github.com/tvgVita69/1C/assets/98489171/e7ca8a52-c7db-4aeb-8986-b62e64903f72)

**Внимание!** Чтобы правильно ввести начальные остатки, необходимо добавить учетную политику на дату перехода на УСН.<br>
Установим рабочую дату – 1 января текущего года.

![image](https://github.com/tvgVita69/1C/assets/98489171/c0e6a7b1-0d0c-49af-8525-b88d637e7164)

**По ссылке «Учет по видам деятельности»:** установим основной порядок отражения авансов «Доходы УСН», в этом случае, при получении аванса от покупателя программа будут регистрировать доход в Книге учета доходов и расходов. <br>Вариант «Доходы комитента» устанавливается, если в организации ведется комиссионная деятельность, и в случае поступления аванса покупателя не будет регистрироваться доход в Книге учета доходов и расходов.<br> 
**Важно!** Чтобы появилась эта опция необходимо в Функциональности на закладке «Торговля» установить флажок в позиции «продажа товаров и услуг комитентов (принципалов).<br>
Развернем ссылку «Порядок признания расходов». Она становится активной, если в качестве объекта налогообложения указан вариант «Доходы минус расходы».

![image](https://github.com/tvgVita69/1C/assets/98489171/29c0d142-78d7-4e3d-a63e-5aac13879a9f)

Т.к. в соответствии с новой редакцией статьи 346.17 НК РФ для признания материальных расходов передача в производство необязательна, то в разделе «Учет расходов» в разделе «Материальные расходы» оставляем только два флажка «Поступление материалов» и «Оплата материалов поставщику».<br>
В соответствии с подпунктом 2 пункта 2 статьи 346.17 НК РФ для признания расходов на приобретение товаров достаточно выполнения трех условий: 

поступления товаров, их оплаты и реализации, что было подтверждено письмом Минфина РФ от 29 октября 2010 г. N 03-11-09/95. Установим в разделе «Расходы на приобретение товаров» только три флажка, т.к. получение дохода (оплаты покупателя) не является обязательным условием при определении налоговой базы.<br>
В соответствии с подп. 8 п. 1 ст. 346.16 НК РФ и письмами письма Минфина России от 28.09.2007 № 03-11-04/2/239, от 29.05.2007 № 03-11-04/2/147 можно списать НДС лишь после учета в составе расходов стоимости приобретенных товаров, работ, услуг, устанавливаем в разделе «Входящий НДС» все флажки.<br>
В разделе «Дополнительные расходы, включаемые в себестоимость» обязательными, являются опции «Поступление доп. Расходов» и «Оплата поставщику». Флажок «списание запасов» устанавливается, только если это прописано в учётной политике организации».<br>
Если организация занимается импортом, то в разделе «Таможенные платежи» устанавливаются все флажки.<br>
Остальные закладки оставляем без изменений.

#### Помощник по переходу на УСН.
Для облегчения перехода в базе на другую систему налогообложения в программе предусмотрен помощник. <br>
**Операции → Изменение режима налогообложения → Помощник по переходу на УСН**<br>
**Важно!** Предварительно установить рабочую дату 31 декабря предшествующего года.

![image](https://github.com/tvgVita69/1C/assets/98489171/55d59264-d94c-44bf-932d-46d5362c478b)

Операции, выполняемые в помощнике, делятся на две группы:

1 группа операций выполняется перед переходом на УСН, 2-ая группа – не6посредственно в момент перехода.<br>
Рассмотрим этапы помощника.<br>
1 группа операций.

![image](https://github.com/tvgVita69/1C/assets/98489171/f38103e6-e1e8-457d-afa6-5abf7a9c140a)

По результатам анализа формируются документы «Списание с расчетного счета» с видом операции «Возврат покупателю», в зависимости от соглашения с покупателем, сумма возврата может быть равной полной сумме аванса или равной сумме НДС.

![image](https://github.com/tvgVita69/1C/assets/98489171/cdfe392e-34e3-4053-86c3-1b74d782ada2)

В открывшемся журнале нажать кнопку «Создать» и ввести документ.
![image](https://github.com/tvgVita69/1C/assets/98489171/3fd59137-5b36-46df-9c24-d1ffbf1e68ac)

Как только будут введены все документы, операция отмечается как выполненная.
![image](https://github.com/tvgVita69/1C/assets/98489171/3c20d282-5373-49a8-a597-d1fa79587ab5)

![image](https://github.com/tvgVita69/1C/assets/98489171/4c79e2b2-0b8a-40c1-96cc-de5655f92162)

Следующим этапом переходим к восстановлению НДС.

После Закрытия месяца делаем щелчок по ссылке «Восстановление НДС», документ формируется автоматически. Чтобы проверить документ, сделать щелчок по ссылке «Восстановление НДС» и выбрать команду «Открыть сформированный документ».

Документы не формируют проводок, только вносят изменения в регистры программы.
